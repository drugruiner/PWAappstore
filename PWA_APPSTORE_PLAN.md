# PWA AppStore-like product notes

## Что можно сделать
- Веб-приложение в стиле App Store (иконки, карточки, экран приложения, отзывы, рейтинги).
- Кнопка «Скачать» с фейковой анимацией загрузки и сменой статусов (`Get` -> `Loading` -> `Open`).
- Поиск, категории, top charts, featured блоки и фильтры.
- Полный PWA: установка на домашний экран, офлайн-кеш, splash, иконки, service worker.

## Ограничения iOS
- PWA на iOS не может полностью повторить нативный App Store (особенно системные анимации и нативные компоненты 1:1).
- Реальных установок приложений из PWA быть не может.
- Нужно избегать нарушения бренда/дизайн-прав Apple: лучше делать «вдохновленный» интерфейс, а не pixel-perfect клон.

## Данные приложений
### Вариант 1: Авто-сбор (парсинг/агрегация)
- Технически возможно собирать метаданные (название, разработчик, рейтинг, иконка, скриншоты, описание) из публичных источников.
- Лучше использовать легальные API/публичные RSS/каталоги, а не агрессивный скрейпинг HTML.
- Важно проверить ToS источников до продакшена.

### Вариант 2: Ручной ввод + авто-добавление
- Можно сделать админ-импорт, куда передаются: `icon`, `name`, `developer` (+ опционально `description`, `rating`, `screenshots`).
- После этого система автоматически создаёт карточку приложения, slug, страницу и индексирует в поиске.
- Подходит как fallback, если парсинг ограничен.

## Рекомендуемая архитектура
- Frontend: Next.js + React + Tailwind.
- PWA: `next-pwa` или ручная настройка service worker + manifest.
- Backend: Supabase/Firebase/Postgres + object storage для иконок/скриншотов.
- Очередь импорта: worker (cron/job), который нормализует данные и обновляет каталог.

## Что можно автоматизировать сразу
- Массовый импорт из CSV/JSON.
- Автогенерация карточек и страниц.
- Валидация полей (дубликаты, длины, формат рейтинга, URL иконки).
- Обновления по расписанию для существующих карточек.
